# CICADA APP BUILD LOOP STANDING RULE

## Standing pattern

For CICADA apps, use:

1. ZIP download at top of ChatGPT response.
2. Completion percentages.
3. PowerShell installer.
4. GitHub Desktop commit/push.
5. ChatGPT GitHub verification.
6. Unreal/local app smoke test.
7. Evidence log.
8. Next phase cluster.

## Why

This avoids:

- manual copy errors
- hidden nested ZIP folders
- ChatGPT direct-write permission limits
- branch drift
- memory loss between chats
- "it worked locally" ghosts
- generated Unreal sludge in GitHub

## Future app default

Unless explicitly changed, future CICADA app builds should follow this loop.

If a better direct-write/repo-agent tool appears later, it can replace the manual push step, but the evidence/phase-gate structure remains.
